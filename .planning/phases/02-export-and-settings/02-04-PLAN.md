  <how-to-verify>
Run the app: `npm run dev` — open http://localhost:5173

**Step 1 — Verify ExportPanel controls appear:**
1. Upload 3 or more PNG/JPG images
2. Confirm ExportPanel is visible in the right sidebar below EditPanel
3. Confirm these controls are present: Resolution dropdown, Frame duration input, Loop toggle, Transition dropdown, "Estimated size" display, "Export GIF" button

**Step 2 — Verify file size estimate is live:**
1. With 3 frames loaded, note the estimated size shown
2. Change resolution to "1200 x 900" — estimated size should increase noticeably
3. Change back to "800 x 600" — estimated size should decrease

**Step 3 — Verify export and download:**
1. Select resolution "800 x 600" (default)
2. Set frame duration to 500ms
3. Ensure Loop is ON
4. Click "Export GIF"
5. Confirm: Progress bar appears and fills from 0% to 100%
6. Confirm: Browser downloads a file named "animation.gif"
7. Confirm: Progress bar disappears after download completes

**Step 4 — Verify the downloaded GIF is correct:**
1. Open animation.gif in a browser tab (drag-drop or File > Open)
2. Confirm: GIF animates (multiple frames play in sequence)
3. Confirm: GIF loops (plays continuously without stopping)
4. Confirm: Frame speed feels roughly 500ms per frame (0.5 seconds each)
5. Confirm: Images appear correctly (letterboxed in the correct resolution)

**Step 5 — Verify loop-off behavior:**
1. Set Loop toggle to OFF in ExportPanel
2. Export again — open the new animation.gif
3. Confirm: GIF plays through once and STOPS on the last frame (does not loop)

**Step 6 — Verify resolution preset works:**
1. Set resolution to "1200 x 900"
2. Export — open the downloaded GIF
3. Right-click the GIF in browser, "Inspect element" or check image properties
4. Confirm: Image dimensions shown as 1200 x 900 pixels

**Step 7 — Verify preview player still works after export:**
1. After the export completes, click Play in PreviewPlayer
2. Confirm: Animation plays correctly (not broken/black)

**Known limitation (acceptable for Phase 2):** On Safari < 17, the export button shows an alert "Export requires a modern browser." This is intentional per research decision — not a bug.
